Obtener sensores activos

Devuelve un array con todos los sensores activos de la organización, incluyendo sus encuestas, páginas y preguntas con sus alternativas.

Método: GET
Endpoint: /api/sensors/

La respuesta es un array de objetos sensor, donde cada sensor incluye:

  • id y name: identificador y nombre del sensor.
  • surveys: array de encuestas del sensor, cada una con sus pages y questions.

Cada pregunta incluye: type, type_str, title, mandatory y alternatives (con nombre, valor e ícono).

Ejemplo con curl:

curl -H "Content-Type: application/json" \
  -H "Authorization: Token 194cd5d43b6a1b6c0dc15403d3eb0adfc80148bf" \
  https://qservus.redcalidad.com/api/sensors/

Recibir respuesta

Registra una o varias respuestas de clientes directamente en el sistema.

Método: POST
Endpoint: /api/responses/

El cuerpo es un array de objetos respuesta, cada uno con:

  • su_survey_id: ID de la encuesta.
  • qs_sensor_id: ID del sensor.
  • creation: fecha y hora de la respuesta.
  • full_name, email, ip, user_agent: datos del respondente.
  • answers: array de respuestas, cada una con su_question_id, answer, answer_motive y answer_other.

La respuesta devuelve un objeto con users_respondents: array de IDs de los respondentes registrados.

Enviar encuesta a destinatario

Envía una encuesta de campaña a un destinatario específico.

Método: POST
Endpoint: /api/send/

El cuerpo incluye los datos del destinatario. La respuesta devuelve un objeto con id_respondent: el ID del destinatario creado en el sistema.

Obtener destinatario en lista negra

Consulta si un correo electrónico está registrado en la lista negra de la organización.

Método: GET
Endpoint: /api/blacklist/

La respuesta es un array de objetos con los destinatarios en lista negra que coinciden con los parámetros de búsqueda.

Verificar estado de la plataforma

Comprueba que los servicios de QServus estén operativos.

Método: GET
Endpoint: /api/status/

La respuesta es un objeto con un atributo que indica el estado actual de la plataforma.

💡
Consulta los Códigos de estado HTTP para interpretar las respuestas de error de la API.
