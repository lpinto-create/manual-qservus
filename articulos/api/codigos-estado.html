Categorías de respuesta

Todos los estados de la API de QServus corresponden a códigos HTTP estándar, agrupados en tres categorías:

  • 2XX — Respuesta exitosa: la solicitud fue recibida y procesada correctamente.
  • 4XX — Error en el lado del cliente: la solicitud contiene un error, datos inválidos o falta de autorización.
  • 5XX — Error en el lado del servidor: el servidor encontró un problema al procesar la solicitud.

Códigos más comunes

CódigoNombreDescripción
200OKLa solicitud fue exitosa y la respuesta contiene los datos solicitados.
201CreatedEl recurso fue creado correctamente.
400Bad RequestLa solicitud tiene un formato incorrecto o le faltan parámetros requeridos.
401UnauthorizedNo se proporcionó un token de autenticación válido.
403ForbiddenEl usuario autenticado no tiene permisos para realizar esta operación.
404Not FoundEl recurso solicitado no existe en el sistema.
500Internal Server ErrorError interno del servidor. Contacta a soporte si persiste.
💡
Si recibes un error 401, verifica que estás incluyendo el token en el encabezado Authorization de la solicitud con el formato correcto: Token <tu_token>.
