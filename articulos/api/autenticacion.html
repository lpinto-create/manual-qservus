¿Cómo funciona?

Para usar la API de QServus debes obtener un token de autenticación. Este token debe incluirse en el encabezado de cada solicitud posterior. La autenticación se realiza enviando una solicitud POST con tus credenciales al endpoint correspondiente.

Parámetros de entrada

El cuerpo de la solicitud debe ser un objeto JSON con los siguientes atributos:

  • email: correo electrónico del usuario registrado en QServus.
  • password: contraseña del usuario.
{
  "email": "valvarez@example.com",
  "password": "333TY2OPZ$"
}

Respuesta

La respuesta es un objeto JSON con los datos del usuario y su empresa, más el token de acceso:

  • id: identificador del usuario.
  • email: correo del usuario.
  • first_name / last_name: nombre y apellido.
  • company: objeto con los datos de la empresa (id, nombre, logo, colores).
  • token: token de autenticación para usar en solicitudes posteriores.
{
  "id": 1,
  "email": "valvarez@example.com",
  "avatar": null,
  "first_name": "Victor",
  "last_name": "Alvarez",
  "company": {
    "id": 1,
    "name": "Example S.A.",
    "logo": "https://example.com/media/logos/5e8c7/085c0.jpg",
    "default_banner": "http://example.com/media/survey_logos/56d8/454.jpg",
    "bg_color": "FFFFFF",
    "font_color": "000000",
    "btn_color": "0000FF",
    "btn_font_color": "FFFFFF"
  },
  "token": "194cd5d43b6a1b6c0dc15403d3eb0adfc80148bf"
}
ℹ️
Incluye el token en el encabezado Authorization de cada solicitud posterior usando el formato: Token 194cd5d43b6a1b6c0dc15403d3eb0adfc80148bf.
